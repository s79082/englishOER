Introduction

Here we want to give you a brief introduction into how the internet works on a day-to-day base. We will explain core concepts and principals as well as some example technologies. Later we will use that knowledge to try to exploit these mechanisms.

A way to the web

Now we will discuss the most common way of accessing the internet. We want to visit a Website using our PC Browser. A browser is a designated program for viewing websites. First we will use the Browser to send a request to a Server. A Server is a powerful Computer which stores our website. So we send the Server a message like "Hey Server, can we please see this Website". If the Server receives that message and answers with "Sure, here you go" and the Website. The website is then rendered by the Browser so we can see a beautiful graphical site.
Because the internet is a web of many computers, we have to make sure the right server receives our request. We need an easy way to address uniquely these computers uniquely to get the wanted response. For this the Internet Protocol (IP) is used. An IP address are 32 bits or four integers between 0 and 255 separated by digits. If you connect to the Internet through an Internet Service Provider (ISP), you are usually assigned a temporary IP address for the duration of your dial-in session. If you connect to the Internet from a local area network (LAN) your computer might have a permanent IP address or it might obtain a temporary one from a DHCP (Dynamic Host Configuration Protocol) server. In any case, if you are connected to the Internet, your computer has a unique IP address. Even though an IP address looks very cryptic one can still find a lot of information about the sender, its location for example. This can be hidden by using an VPN

Try it! - You can check if an computer is connected to the internet using the ping command. Open a terminal(Linux) or a command prompt (Windows) and type in
ping htw-dresden.de
press enter. The program will send messages to the Server and show the responses. You can also see the IP Address of the URL.

The OSI protocol stack

Of course the process of sending and receiving data is a little bit more complex than described before. Introducing the Open Systems Interconnection model (OSI model). A (communication) Protocol describes the communication via the internet. The OSI model is a stack of these protocols.
In this image you can see the process of sending a message from your computer (client) to another computer (server). The data package starts at the top of the left stack in the upper most layer, the Application layer. It makes its way down through the layers below it. Each layer has its own function and adds information to the package or splits the package accordingly to its function. At the Hardware layer the data gets transmitted via electrical currents and is been send through wires and radio waves to the physical layer of the receiver, where the package makes its way through the stack to the application layer. So even though the packet hast to go through all the layers you can imagine them logically communication in one layer. The layers you see are the Hardware (physical) layer where the bits are transformed into physical signals. The IP layer is responsible for address resolution. In the Application layer lives the Browser who processes and renders websites. Here it is where the HTTP protocol lives.

The Application Layer

The application layer is the top most layer and its the most abstract layer. This makes it the most versatile but fragile of all layers. One of the most commonly used services on the Internet is the World Wide Web (WWW). The application protocol that makes the web work is Hypertext Transfer Protocol or HTTP. Do not confuse this with the Hypertext Markup Language (HTML). HTML is the language used to write web pages. HTTP is the protocol that web browsers and web servers use to communicate with each other over the Internet. It is an application level protocol because it sits on top of the TCP layer in the protocol stack and is used by specific applications to talk to one another. In this case the applications are web browsers and web servers. HTTP is a connectionless text based protocol. Clients (web browsers) send requests to web servers for web elements such as web pages and images. After the request is served by a server, the connection between client and server is disconnected. A new connection must be made for each request. Most protocols are connection oriented. This means that the two computers communicating with each other keep the connection open over the Internet. HTTP does not however. Before an HTTP request can be made by a client, a new connection must be made to the server. When you type a URL into a web browser, this is what happens:
  1. If the URL contains a domain name, the browser first connects to a domain name server and retrieves the corresponding IP address for the web server.
  2. The web browser connects to the web server and sends an HTTP request (via the protocol stack) for the desired web page.
  3. The web server receives the request and checks for the desired page. If the page exists, the web server sends it. If the server cannot find the requested page, it will send an HTTP 404 error message.
  4. The web browser receives the page back and the connection is closed.
  5. The browser then parses through the page and looks for other page elements it needs to complete the web page. These usually include images, applets, etc.
  6. For each element needed, the browser makes additional connections and HTTP requests to the server for each element.
  7. When the browser has finished loading all images, applets, etc. the page will be completely loaded in the browser window.

DOM and JavaScript

So we have already shown you how to receive a website on your PC. But this process is very static: once the server has send a response, there is no way changes to the website can be made. This is really bad for things like browser games where you want some game logic executed clientside. To change the contents of a site dynamical we need JavaScript and the Domain Object Model. DOM is a hierarchical, tree-like struct which represent elements on our website. DOM is language-independent which means that a list element has both a representation in HTML (the "<ul>"-tag) and as an object in javascript

Try it!- Lets change the site a little bit. Hover over the headline of this paragraph, right-click and select inspect. You will see a HTML tag called h3, a type of headline. You will also see an attribute to this headline called id. This is the identification of our DOM object. We can use this id to get a hold of our headline:
Type in your console:
myheadline = document.getElementById("myheadline")
This will asign the DOM Object of this headline to a Javascript variable called myheadline. We can now use this variable to modify the object. Type
myheadline.innerHTML = "This is my new Headline :)"
Press enter and look what happend to the old headline
We can also add completely new object to our site. Try:
mytext = document.createTextNode("hello Im a text node")
to create a text node. Then type
document.body.appendChild(mytext)
to add your text at the bottom of the site.
